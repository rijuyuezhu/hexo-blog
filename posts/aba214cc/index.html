Posted on 2024-03-19 In Course Note , OS

When using MPE part in x86-64 qemu in Abstract Machine of NJU-ProjectN, the program did not run normally: only one CPU is avaliable though we set smp to 4.

Problem Description

The problem occurred when we attempting to run the demos of Prof. Jiang's Operating System in Nanjing Univ.. More specifically, the demo of spinlock( lecture link, code link).

When running the code, after setting appropriate AM_HOME etc, we found that only one CPU is on:

But we set smp to 4 indeed:

Using gdb, we found that the AM API cpu_count() returns 1, but this contradicts the smp argument we set. We guessed that this problem results from a combination of the qemu version and the problems inside $AM_HOME/am/src/x86/qemu/trm.c: __am_lapic_init(). The latter is designed to load MP table and so on, and initialize __am_ncpu. Then cpu_count() returns __am_ncpu.

We failed to find the problem so far. However, some solutions are provided.
